Description and Demonstration :

- CSS uses the @keyframes rule to animate elements in HTML

- the @keyframes rule describes the animation steps (protocol) to animate the referenced element 

SYNTAX :    @keyframes animationName {
                from / 0% {CSS CODE}
                  animation steps...
                to / 100% {CSS CODE}
            }    
                - from / to / 0% - 100% : "keyframes-selectors" select the animation steps
    
    <p class="myClass" > myParagraph </p>
    
-- CSS --
    .myClass {
        animation: myAnimation 4s infinite;                     // animation protocol is called here ("myAnimation")
    }
    @keyframes myAnimation {                                    // specifies the animation steps (the "myAnimation" is the animation name)
        from {color: black;}
        to {color: red;}
    }

animation-name / animation-duration / animation-delay / animation-iteration-count 

SYNTAX :    animation-name: none            // (default) no animation bound         //  references the @keyframes animation
                          : animName 
                          
            animation-duration: nrs/ms      // default is 0                         //  specifies the length of one animation cycle    
            
            animation-delay: nrs/ms         // default is 0, negative values are allowed      //  sets a delay for the animation start
            
            animation-iteration-count: nr   // default is 1                         //  specifies the number of times the animation is played
                                     : infinite
    
    <p class="myElement"> myParagraph </p>
    
-- CSS --
    .myElement {
        position: relative;
        animation-name: myAnimation;                                                 // references the keyframe id
        animation-duration: 10s;
        animation-iteration-count: infinite;
        animation-delay: 2s;                                                         // 2 seconds before the animation starts (if value is negative the animation is handeled as it is already playing for the set time)  
    }
    @keyframes myAnimation{
        from {left: 0;}
        to {left: 200px;}
    }

                               : alternate-reverse  // end-start-end

animation-timing-function 

SYNTAX :    animation-timing-function: ease                         // (default)     //  specifies the speed curve of the animation  
                                     : linear 
                                     : ease-in  
                                     : ease-out  
                                     : ease-in-out  
                                     : step-start                   // same as step(1, start)
                                     : step-end                     // same as step(1, end)
                                     : steps(nr, start|end)         // breaks the fluid animation into steps (2nd parameter default end = animation starts from step 0 | start animation starts from step 1)
                                     : cubic-bezier(nr, nr, nr,nr)  // sets a custom animation speed curve cubic-bezier generator [link]                      
    
    <p class="demo"> I rotate</p>
    
-- CSS --
    .demo {
        width: 150px;
        text-align: center;
        animation-direction: alternate;
        animation-iteration-count: infinite;
        animation-duration: 2s;
        animation-timing-function: cubic-bezier(.03, .68, 1, .61);                   // custom speed curve used
        animation-name: rotateMe;
    }
    @keyframes rotateMe {
        from {transform: rotateZ(0)}
        to {transform:rotateZ(75deg)}
    }

animation-fill-mode

- the animation-delay period is technically animation already, this is where the animation-fill-mode:backwards commes into play 

SYNTAX :    animation-fill-mode: none               // (default) no style applied    //  specifies the style for the element when the animation is fired but NOT playing (delayed, finished)
                               : forwards           // the last @keyframes style is preserved when animation is finished
                               : backwards          // the first @keyframes style is preserved when animation starts (delayed period) 
                               : both               // both the first and last @keyframes style are preserved (animation start (delayed period) / animation finished)
    
    <p class="fillMode"> Fill Moded </p>
    
-- CSS --
    .fillMode {
        position: relative;
        left: 0;                                    // element never uses its original position in this case!
        animation-duration: 5s;
        animation-delay: 3s;                        // during the delayed period the "left: 50px" is applied because of the "animation-fill-mode: both;"  
        animation-name: fillMe;
        animation-fill-mode: both;                  // both the first and last keyframes styles are applied
    }
    @keyframes fillMe {
        0% {left: 50px;}
        100% {left: 300px;}                         // at the end of the animation this style is preserved because of the "animation-fill-mode: both;"
    }

TEST

- note that : the animation has a 2 seconds delay before it starts, that is technically a running animation

- elements are set to left:50px; / animation start at left:0; finished left:500px;

animation-fill-mode: forwards

- first keyframe style is NOT applied, during the animation delay period the element stays on its original position left:50px;

- last keyframe left:500px; is applied when animation is finished

animation-fill-mode: backwards

- first keyframe left:0; is applided during the animation delay period

- the last keyframe style is NOT applied, when animation is finished (element jumps to its original position left:50px;)

animation-fill-mode: both

- first keyframe left:0; is applided during the animation delay period

- last keyframe left:500px; is applied when animation is finished
