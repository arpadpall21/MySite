Description and Demonstration:

- JavaScript is an OOP (Object Oriented Programing) language officialy ECMA-262 which mostly power web browsers (Not Procedural Programing Language)

- In HTML JavaScript must be inserted between <script> and </script> tags

- JavaScritp is a single threated language (does 1 thing at the time only) which manages its memory (high-level language)

- JS executes tasks what are queued in the "Call Stack"

- JS send tasks to APIs (they opperate on their own thread!), once the API finishes it puts its callback function in the "Callback Queue"

- the "Event Loop" opperates between the "Call Stack" and the "Callback Queue", if the "Call Stack" becomes empty the "Event Loop" moves the queued callback function form the "Callback Queue" to the "Call Stack" this is 1 Tick, what is executed by JS, than the process repeats until nothig left to execute

- a Promise response is executed at the end of the next Tick called "Job Queue"

Statements and Expressions

- a JavaScript statement is basically a piece of instruction to the computer

- a JavaScript expression is a valid unit of code that resolves to a value (side effect or returned value) 

    var x;                                               // this is a statement (instruction to the computer)
    12;                                                  // this is an expression because it produces a value 
        
// ----------------------------------------------------- 
    if ( x < y ) {                                       // the if conditional is a statement (instruction to the computer)
        document.write("You're awesome!");               // the "x < y" is an expression because it will produce a value 
    } else {
        document.write("You're not awesome!");
    }

Scoping

- the purpose of scopes in JavaScript is to determinate the accessibility (visibility) of variables

- in JavaScript there are two types of scope : "Global Scope" global object of the host (declared variables accessible anywhere) and "Local Scope" function scope (declared variables are NOT accessible from outside)

- identifiers declared without any "variable keyword" will become automatically global variables (but this practice fails in "strict mode"!) 

// this is the "Global Scope" (basically this is the "window" object)
    var x = 10;                                          // this is a global variable 
    var y = 11;
    
    x;                                                   // -> 10                           // global variable called form the global scope
    a;                                                   // -> undefined                    // -! form the global scope we cannot access any local scopes
    b;                                                   // -> 40                           // automatically global variable called (no problem)
    
// we create lexical environment here but normally function variables only lives during the function call 
// this is a "Local Scope" (environment inside a function)
    (function() {
        var x = 20;                                      // this is a local variable        // -! it is not the same as the "x" on the global scope!
        var z = 21;
        
        y;                                               // -> 11                           // global variable called form a local scope
        x;                                               // -> 20                           // we call the variable from this local scope (not the one on the global scope)
        a;                                               // -> undefined                    // -! local variables nested UNDER are NOT accessible 
        
// this is also a "Local Scope" (nested in another function) 
        (function() {
            var a = 31;                                  // this is a local variable 
            b = 40;                                      // !! this is a global variable because it is declared withouth any variable keyword (in "strict mode" this fails!)   
            
            a;                                           // -> 40                           // call a variable from this local scope 
            z;                                           // -> 21                           // -! local variables ABOVE are accessible 
            y;                                           // -> 11                           // any global variables can be accessed from anywhere 
        })();
    })();

Hoisting

JavaScript moves all declarations to the top of the current scope but not initializations

Always declared and initilizate your variables on the top of your code this prevent confusion of Hoisting

    var a;                                      // this is a declaration;
    b = 2;                                      // this is an initialization;
    var c = 3;                                  // this is an initialization too;       // ALWAYS USE THIS!;
    
    window.write(
                a + "<br> +                     // will return undefined (because the initialization is at the bottom of the code)
                b + "<br> +                     // will return 2 (because the declaration is moved to the top) 
                c;                              // will return 3 (because it was initializated on the top)
                );
                   
    a = 1;                                      // initialization (not moved to the top)
    var b;                                      // declaration (moved to the top)

Strict Mode

- JavaScript strict mode is a mode to better check for errors, it basically turns "bad syntax" into real errors 

SYNTAX :    "use strict"                                                               // 5 enables strict mode at the beginning of the current scope
                                                                                          undeclared variables are not allowed (var keyword required)   
                                                                                          deleting variables, objects or functions with the delete keyword is not allowed 
                                                                                          deleting undeletable property is not allowed (ex: delete Object.prototype)    
                                                                                          duplicate arguments is not allowed (ex: function(a, a){})   
                                                                                          octal literals or octal escape characters are not allowed (ex: 010 / '\045')    
                                                                                          set or get only properties are not allowed 
                                                                                          using the eval() method to create variables is not allowed 
                                                                                          using the with() method is not allowed
DEMO 
    "use strict";                               // strict mode in global scope (local sopes are running in strict mode to in this case!)
    
    function myFunc() {
        "use strict"                            // using strict mode in local scope (in this case the global scope is NOT running in strict mode)
    }

- words not allowed as variable names in strict mode 

    arguments       interface       private         static
    eval            let             protected       yield
    implements      package         public

- undeclared variables will cause error in Strict Mode 

    x = "this string";                          // throws an error in strict mode
    y = {Mark:"Audi", Model:"A8"};              // throws an error in strict mode

- deleting variable, object or function is Not Allowed in Strict Mode 

    var x = 21;
    var y = {p1:10, p2:20};
    function z(a, b){};
    
    delete x;                                   // throws an error in strict mode
    delete y;                                   // throws an error in strict mode
    delete z;                                   // throws an error in strict mode

- duplicating parameters is Not Allowed in Strict Mode

    function myFunc(a, a){}                     // throws an error in strict mode

- octal literals 010 and octal escape characters \"010 are Not Allowed in Stirct Mode 

    var x = 010;                                // throws an error in strict mode (octal literal)
    var y = "\045";                             // throws an error in strict mode (octal escape literal)

- set read-only and get-only properties are Not Allowed in Strict Mode 

// read-only property -------------------- 
    var myObj = {prop1:"myVal"};
    Object.defineProperty(myObj, "prop1", {writable:false});    // this won't throw an error (it is OK to have read-only property)
    myObj.prop1;                                // -> "myVal"   // we can read read-only property in strict mode no problem!
    myObj.prop1 = "modifiedVal";                // throws an error in strict mode (read-only property cannot be modified)
    
// get-only property ---------------------
    var myObj = {get prop1() {return "value what you get"}};    // the "prop1" is set as getter-only propearty (no "set" specified)
    myObj.prop1;                                // -> "value what you get"      // we can read the get-only property no problem!
    myObj.prop1 = "set new value";              // throws an error in strict mode (because the property has no setter function)

- deleting an undeletable property in Strict Mode is Not Allowed 

    delete Object.prototype;                    // throws an error in strict mode

- using the eval() function to create variables is Not Allowed in Strict Mode 

    eval("var x = 'this variable'");            // throws an error in strict mode

- using the with statement is Not Allowed in Strict Mode 

    with (Math){                                // throws an error in strict mode  
        x = cos(3 * PI) + sin (LN10)    
        y = tan(14 * E)  
    }
