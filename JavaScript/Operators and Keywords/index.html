Operator Precedence

- operator precedence describes the order in wich order the operations will be performed in an arithmetic expression (like the * and / have higher precedence than the + and -)

- equal precedence operators are computed form left to right

    var x = 100 + 50 * 3;                       // -> 250   // the multiplication have higher precedence
    var x = (100 + 50) * 3;                     // -> 450   // precedence changed
    var x = 100 + 50 - 3;                       // -> 147   // equal precedence operators are computed from left to right

Unary Operators 

SYNTAX :    ++variable                          // "prefix increment" increases the value and returns it
            --variable                          // "prefix decrement" decreases the value and returns it
            variable++                          // "postfix increment" increases the value but does not return it
            variable--                          // "postfix decrement" decreases the value but does not return it
            
            +value                              // converts the value into a number
            -value                              // converts the value into a number and flips [Negative to Positive] / [Positive to Negative]
    
    var myNumber = 10;
    ++myNumber;                                 // -> 11        
    --myNumber;                                 // -> 9
    myNumber++;                                 // -> 10        // value modified but NOT returned
    myNumber--;                                 // -> 10        // value modified but NOT returned
    
//  --------------------------------------------    
    +"21";                                      // -> 21        // values are converted to numbers
    +"-21";                                     // -> -21
    
    -"55"                                       // -> -55       // value converted to number and flipped form positive to negative 
    -"-55"                                      // -> 55        // value converted to number and flipped form negative to positive

Assignment Operators 

SYNTAX :    identifier = value                  // assigns a value to an identifier

            variable += value                   // adds a value to the variable and returns it
            variable -= value                   // subtracts a value from the variable and returns it
            variable *= value                   // multiplies the variable by the value and returns it
            variable /= value                   // divides the variable by the value and returns it
    
            variable **= value                  // raises the variable to the power of the given value and returns it
            variable %= value                   // modulo divides the variable by the given value and returns it
            
            variable <<= value                  // "bitwise operator" left shifts (zero fill) the variable by the passed value and returns the result
            variable >>= value                  // "bitwise operator" right shifts (preserve shift-fill) the variable by the passed value and returns the result 
            variable >>>= value                 // "bitwise operator" right shifts (zero fill) the variable by the passed value and returns the result 
            variable &= value                   // "bitwise operator" uses the "AND" bitwise operator on the variable with the passed value and returns the result
            variable |= value                   // "bitwise operator" uses the "OR" bitwise operator on the variable with the passed value and returns the result
            variable ^= value                   // "bitwise operator" uses the "XOR" bitwise operator on the variable with the passed value and returns the result
    
    var myVariable = 10;
    
    myVariable += 2;                            // -> 12                // same as ->       myVariable = myVariable + 2;
    myVariable -= 2;                            // -> 8                 // same as ->       myVariable = myVariable - 2;
    myVariable *= 2;                            // -> 20                // same as ->       myVariable = myVariable * 2;
    myVariable /= 2;                            // -> 5                 // same as ->       myVariable = myVariable / 2;
    
    myVarialbe **=3;                            // -> 1000              // same as ->       myVariable = myVariable ** 3;
    myVariable %=3;                             // -> 1;                // same as ->       myVariable = myVariable % 3;
    
// bitwise assignement operators ---------------------------------    
    myVariable <<= 2;                           // -> 80;               // same as ->       myVariable = myVariable << 2;
    myVariable >>= 2;                           // -> 2;                // same as ->       myVariable = myVariable >> 2;
    myVariable >>>= 2;                          // -> 2;                // same as ->       myVariable = myVariable >>> 2;
    myVariable &= 2;                            // -> 2;                // same as ->       myVariable = myVarialbe & 2;
    myVariable |= 2;                            // -> 10;               // same as ->       myVariable = myVarialbe | 2;
    myVariable ^= 2;                            // -> 8;                // same as ->       myVariable = myVariable ^ 2;

Comparison Operators

- comparison operators compare 2 values and return a boolean result true or false

- comparison operators compare primitive values, therefore 2 objects cannot be compared (returns false) 

SYNTAX :    value == value                      // "abstract equality comparison", returns true if the values are equal (no data type test)
            value === value                     // "same value comparison", returns true if the values and the datatypes are equal 
            value != value                      // "abstract equality comparison not" returns true if the values are not equal (no data type test)
            value !== value                     // "same value comparison not" returns true if the values and the datatypes are not equal
                
            value < value                       // "less than" returns true if the left value is lower (no data type test)
            value <= value                      // "less or equal than" returns true if the left value is lower or equal (no data type test)
            value > value                       // "greater than" returns true if the left value is greather (no data type test)
            value >= value                      // "greater or equal than" returns true if the left value is greater or equal (no data type test)
    
    myObj == myObj                              // -> false             //objects cannot be compared! always "false"
    "5" == 5                                    // -> true
    "5" === 5                                   // -> false             // datatypes are not equal
    "5" != 5                                    // -> false             // both values are equal
    "5" !== 5                                   // -> true              // datatypes are not equal
    
    "3" < 5                                     // -> true              // no datatype test
    "5" <= 5                                    // -> true              
    "3" > 5                                     // -> false              
    "5" >= 5                                    // -> true
    
    "abc" < "def"                               // -> true              // strings are compared in Alphabetical Order 
    "apa" < "anya"                              // -> false
    "a" > 15                                    // -> false             // letter compared to a number always "false"
    "a" > "15"                                  // -> true              // numbers are placed before letters when it comes to an Alphabetical comparison

Logical Operators

- logical operators try to convert expression into boolean and act accordingly 

SYNTAX :    exp_1 && exp_2                      // "logical AND" if the 1st expression is "false" it returns the 1st exression (and never evaluates the 2nd one [Short-Circuits]), otherwise returns the 2nd expression  
            exp_1 || exp_2                      // "logical OR" if the 1st expression is "true" it returns the 1st expression (and never evaluates the 2nd one [Short-Circuits]), otherwise returns the 2nd expression  
            !exp                                // "logical NOT" if the expression is "ture" it returns false, otherwise returns true
    
    "myString" && undefined;                    // -> undefined     // 1st exression is "true" it returns the 2nd one
    null || 66;                                 // -> 66            // 1st expression is "false" it returns the 2nd one
// SHORT-CIRCUITING ------------------------    
    undefined && ImaFucking_Virus               // -> undefined     // 1st expression is "false" so it is returned
                                                                    //2nd expression (ImaFucking_Virus) is not even evaluated [an exception should be trown but nothing happens = SHORT-CIRCUITING]  
    21 || ImaFucking_Virus                      // -> 21            // 1st expression is "ture" so it is returned
                                                                    //2nd expression (ImaFucking_Virus) is not even evaluated [an exception should be trown but nothing happens = SHORT-CIRCUITING]  
// logical NOT -----------------------------
    !21                                         // -> false         // expression can be evalueate "true"
    !null                                       // -> true          // expression cannot be evaluated "true"

Bitwise Operators

- JavaScript stores numbers in 64 bits floating point numbers, but when it comes to bitwise operation JS converts them into 32 bits binary signed integers, after the operation numbers are converted back to 64 bits floating point again

- bitwise operators work with the binary representatinon of a number not with the number's value, (threat a set of bits instead of a single number) 

SYNTAX :    nr & nr                                                                    // "bitwise AND" sets the current bit in the sequence to 1 if both bits are 1
            nr | nr                                                                    // "bitwise OR" sets the current bit in the sequence to 1 if one of the two is 1 or both are 1
            nr ^ nr                                                                    // "bitwise XOR" sets the current bit in the sequence to 1 if one of the two bits is 1
            
            ~nr                                                                        // "bitwise NOT" inverts all the bits
            
            nr << nr                                                                   // "bitwise Left Shift Zero-Fill" shifts the bits of the left operand to left specified by the right operand and fills the new space with 0
            nr >>> nr                                                                  // "bitwise Right Shift Zero-Fill" shifts the bits of the left operand to right specified by the right operand and fills the new space with 0
            nr >> nr                                                                   // "bitwise Right Shift Preserve-Fill" shifts the bits ot the left operand to right specified by the right operand and fills the new space by using the current front bit     
    
// bitwise AND ------------------------------------------------------------------------
    21 & 68;                                                                           // -> 4      
        // 21 (decimal) = 00000000000000000000000000010101 (32 bits binary)   
        // 68 (decimal) = 00000000000000000000000001000100 (32 bits binary)
        //  4 (decimal) = 00000000000000000000000000000100 (32 bits binary)            // sets the bit to 1 if both are 1
                 
// bitwise OR -------------------------------------------------------------------------
    21 | 68;                                                                           // -> 85       
        // 21 (decimal) = 00000000000000000000000000010101 (32 bits binary)   
        // 68 (decimal) = 00000000000000000000000001000100 (32 bits binary)
        // 85 (decimal) = 00000000000000000000000001010101 (32 bits binary)            // sets the bit to 1 if one of the two is 1 or both are 1 
                
// bitwise XOR ------------------------------------------------------------------------           
    21 ^ 68;                                                                           // -> 81

        //  21 (decimal) = 00000000000000000000000000010101 (32 bits binary)                   
        //  42 (decimal) = 00000000000000000000000000101010 (32 bits binary)           // all bits are shifted to left by 1 and the added bit is 0

declaring identifiers var / let / const

SYNTAX :    var identifier = value              // declares a hoisted variable (Reassignable / Redeclarable) (registered in the ObjectEnvironmentRecord = creates properties on the current scope)
            let identifier = value              // declares a non-hoisted block-scope {} local-variable (Reassignable / NOT-Redeclarable) (registered in the DeclarativeEnvironmentRecord = does NOT create properties on the current scope (registered-only)) (new in ES6)   
            const identifier = value            // declares a non-hoisted block-scope {} local-variable (NOT-Redeclarable / NOT-Redeclarable) (registered in the DeclarativeEnvironmentRecord = does NOT create properties on the current scope (registered-only)) (new in ES6)    
    
    if (true) {                                 // this is a block scope not a local scope! 
        var x = 10;                             // this { } is considered as a block scope  
        let y = 11;
        const z = 12;
    }
    x;                                          // returns "10", "var" value can be accessed outside the code block
    y; / z;                                     // throws an Error because the "let" and the "const" only available inside the code block                       
    
    var x = 10;
    x = 11;                                     // can be reassigned;
    var x = 12;                                 // can be redeclared;
    
    let y = 20;
    y = 21;                                     // can be reassigned;
    let y = 22;                                 // cannot be redeclared (Throws an Error)
    
    const z = 30;
    z = 30;                                     // cannot be reassigned (Throws an Error)
    const z = 32;                               // cannot be redeclared (Throws an Error)

- the let and the const keywords does not create a global object properties like the var does (JavaScript keeps these identifiers in the DER (declarativ environment record) 

    var x = 10;         
    window.x                                    // -> 10
    x                                           // -> 10
//  ------------------------------------
    let y = 11;
    window.y                                    // -> undefined (not a property of the current scope)
    y                                           // -> 11
//  ------------------------------------
    let z = 12
    window.z                                    // -> undefined (not a property of the current scope)
    z                                           // -> 12

- the var keyword is hoisted (declaration moves to the top) VS the let and const are not hoisted!

    x = 10;
    var x;                                      // "x" is hoisted 
    
    y = 11;
    let y;                                      // "y" is not hoister (Throws an Error)  
    
    z = 11;
    const z;                                    // "z" is not hoister (Throws an Error)

- interestingly we cannot redeclare the let / const or reassign the const but if we use them on an Object or an Array we still can modify them by using some methods 

    const myArray = [];                         // cannot be redeclare or reassign 
    myArray.push("element");                    // add a new element through a method
    
    myArray[0];                                 // -> "element"
