Description and Demonstration : 

- tool for application in an isolated environment 
- the app runs in the same environment 
    
- Docker container packages the code and dependencies together 
    - multiple containers can be run on the same machine 
    
- Docker VS Virtual Machine
    - Docker run apps VS Virtual Machin runs a full OS
    - We don't know to run a full OS we just need Docker which runs apps in contaier 
    
- Docker image (snapshot) is a file used to execute code in a Docker container 
    - image is a set of instruction to build a docker container 
    - from an image you can build a container 
    
    
-- commands ----------
    - ps [-a]                                       -> prints the running containers  
                                                        -a -> prints all containers 
    
    - container ls              
    
    - pull image                                    -> pulls an image 
    
    - images                                        -> list of docker images 
    
    - run [-d] [-p srcPort:contPort] [--name contName] [-v srcDir:trgDir[:ro]] [--volumes-from contName] image:tag
                                                    -> runs the specified image as a container 
                                                        -d -> detached mode  
                                                        -p srcPort:trgPort -> port mapping (multiple ports can be mapped)
                                                        --name name  -> sets the name of the container 
                                                        --volumes-from contName -> shares the volume from the source container to this newly created container   
    
    - start contName                                -> starts the specified container 
    
    - stop containerId|contName                     -> stops the specified container 
    
    - rm [-f] contId|contName ... | $(docker ps -aq)  -> removes the sepcified container(s) | removes all containers 
                                                        -f -> forcefully removes containers (even running onces => uses SIGKILL)
    
    - exec -it contName bash                        -> opens the container (bash runs inside the container) (must be a running container)  
    
    
    
    
- volume
    - shares data between host and container or between containers 
    
    
- Dockerfile
    - create a 'Dockerfile' in the directory (context)
    - file content is caseinsensitive but the convertion is that instructions are written with UPPERCASE 
    
    
    
    
    - in file
        # comment 
        FROM image:tag                  // from which image we are building (must be the first command)   
        ADD src trg                     // add a volume to the Dcoker image 
        WORKDIR dirPath                 // sets the working directory for any RUN, CMD, ENTRYPOINT, COPY, ADD instructions that follow int hte Dockerfile  
        RUN cmd                         // runs the command when building a the image 
        CMD cmd                         // this command will be executed when the container starts 
    
    
    - build [-f path/Dockerfile] [t trgDir] [--tag imageName:tagName]  -> builds an image from the Dockerfile and a contxt
                                                            -f path/Dockerfile builds from this source Dockerfile    
                                                            -t target directory where to build the new image    
                                                            --tag 
                                                            
                                                            
- dockerignore                          // files or directories specified here will be excluded when building the docker image   
    filePath|dirPath
    
    
- in order to use the BuildKit (supported from 18.09) set the DOCKER_BUILDKIT=1 before you build the image
