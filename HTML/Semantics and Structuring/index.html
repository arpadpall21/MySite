<bdo> / <bdi>

- in the Unicode table every character has its own defined writing direction (except neutral ones), the browsers uses these information to define the writing direction

- neutral characters like "!.()" do not have writing directions, they use the writing direction of the characters what surround them

- we can use the dir global attribute to define the display direction of any element but not the writing direction, (neutral characters apply the "dir" writing direction) carefull!

- the <bdo> tag allows to change the writing direction VS the <bdi> tag isolates characters from this effect, characters inside it strictly use their original direction from the Unicode 

SYNTAX :    <bdo>                                 // (Bi-Directional Override) overrides the characters  writing direction (ex: Kicsi Kutya)
            <bdi>                                 //  (Bi-Directional Isolation) isolates a text's directions from the rest of the page, characters will use their own Unicode defined direction regardless of the page settings (because of this reason the "dir" attribute is not supported)   
    
    // <bio> element attributes - - - - - 
            dir="ltr|rtl"              // text direction (required global attribute)   
    
    <p> love حب </p>                              // in this paragraph we have 2 writing directions (no need to set these the browser uses the associated character direction of the Unicode character table)  
    <p dir="rtl"> love حب szeretet </p>           // this paragraph will be displayed from right to left no writing direction changed!
    
// <bdo> --------------------------    
    <p><bdo dir="rtl"> love حب szeretet </bdo></p>  // displays "love حب szeretet" the "<bdo>" tag can change the writing-direction to all characters
    <p><bdo dir="ltr"> love حب szeretet </bdo></p>  // displays "love حب szeretet" 
    <p><bdo> love حب szeretet </bdo></p>            // displays "love حب szeretet"  <-- this dose the same because the browser default writing directions is "left-to-right
    
// <bdi> --------------------------
    <bdi dir="rtl"> love حب szeretet </bdi>        // displays "love حب szeretet", the Unicode writing direction of the characters are respected in any situation ("dir" attribute is not supported)
    <p><bdo dir="rtl">paragraph <bdi>isolated bi-direction</bdi> paragraph </bdo></p>   // displays "paragraph isolated bi-direction paragraph"

<meter> / <progress>

<meter> tag

- if the "min" attribute is omitted it is set to "0" (min='0') by default

- the "optimum" attribute defines the optimal range (not the targeted value) (ex: range from "min" to "low" | "low" to "high") by targeting a value between those thresholds

- if you define thresholds with the "low" and "high" attributes, it is highly recommended to use the "optimum" attribute to define the optimal range (otherwise it produces weird color threshold results [TESTED 2019-01-31])

- we can style things like size border, but gauge meter styling is a pretty advanced stuff and not well supported, we use special pseudo-classes, you can read about here: CSS-tricks meter element 

SYNTAX :    <meter>                               //  defines a gauge (scalar measurement or fractional value)
            <progress>                            //  represents a progress of a task
            
    // <meter> element attributes - - - - -         
            min="nr"                   // minimal value (if omitted is 0) 
            low="nr"                   // low threshold (value excluded)
            optimum="nr"               // sets the optimal range by targeting a value between a range (recommended if you want to use thresholds in you gauge)   
            high="nr"                  // high threshold (value excluded)
            max="nr"                   // maximal value
            value="nr"                 // (required) current value 
            form="formId"              // form(s) where the gauge belong (s) to (browser support not TESTED)
            
    // <progress> element attributes - - - - -         
            max="nr'                   // how much is the total work
            value="nr"                 // how much is completed
            
    
// <meter> ----------------------------------------
    <meter max="10" value="7"> 7 out of 11 </meter>                                         // displays 7 out of 10
            // the "7 out of 10" (text between tags) is used as fallback system | because the "min" value is not specified is set to "0" by default  
            
    <meter min="1" low="3" optimum="5" high="7" max="10" value="6"> 6 out of 10 </meter>    // displays 6 out of 10
            // optimal value is between "low" and "high" value at that range is green | "3" and "7" are included in the optimum value
    <meter min="1" low="3" optimum="5" high="7" max="10" value="2">2 out of 10</meter>      // displays 2 out of 10
            // value is not in the optimal range gauge turns yellow
    <meter min="1" low="3" optimum="5" high="7" max="10" value="8">8 out of 10</meter>      // displays 6 out of 10
            // value is not in the optimal range gauge turns yellow
    <meter min="1" low="3" optimum="2" high="7" max="10" value="8">8 out of 10</meter>      // displays 8 out of 10
            // optimal value is between "min" and "low" | "min" and "low" = green | "low" and "high" = yellow | "high" and "max" = red
            
// <progress> (JS controls it in real code) -------
    <progress max="100" value="41">41 out of 100 completed</progress>                       // displays 
            // the "41 out of 100 completed" text between tags is used as fallback system
    <progress max="100" value="94">94 out of 100 completed</progress>                       // displays

SYNTAX :    <dialog>                                  // .2 defines a dialog box (Modal) (as wide as it's content, centerd and absolute positioned box)

            open  (boolean)            // shows/hides the dialog box

    --- JS ---
            HTMLDialogElement.show()                  // opens the dialog box (same as open attribute) (Chromium Only! [2019-02-01])
            HTMLDialogElement.showModal()             // opens the dialog box with backdrop (user unable to click on other elements, closable by the "Esc" button) (Chromium Only! [2019-02-01])   
            HTMLDialogElement.close("message")        // closes the dialog box (same as open attribute) (Chromium Only! [2019-02-01]) 
            HTMLDialogElement.returnValue             // returns the message passed by the close() method (Chromium Only! [2019-02-01])

    --- CSS --- 
            element::backdrop                         // selects the backdrop (box rendered beneath a "full screen-mode element")(usually <dialog> element)  
    
    <dialog class="myModal"> Hello I'm a Dialog Box </dialog>
<script>
JavaScript only works in Chromium [TESTED 2019-02-01]
    var modal = document.querySelector("dialog");

HTML5 Semantic Elements

- Semantic elements are elements with meaning (describles its meaing both for the browser and for the devloper (like : <form> / <table>)

<header> / <nav> / <main> / <article> / <section> / <aside> / <footer>

- there is no rule or definition how these elements nest each other, it is up to the developer and the current page structure how we use them (like the "article" element can nest "section" elements or vice versa)

SYNTAX :    <header>                              //  specifies a header (introductory content) for the document or article
                                                     - multiple allowed on a page 
          ----
            <nav>                                 //  specifies a navigation block within the current document or other documents 
                                                     - intended to wrap major navigation blocks
                                                     - random links on the page are not necessarily wrapped  
          ----    
            <main>                                //  describes the main content of the page 
                                                     - only one "main" element is allowed on a page 
                                                     - it should NOT contain "repeated content" such as navigation bar, copyright info, logos ect.. 
                                                     - in must NOT be a descendent of <article> / <aside> / <footer> / <header> / <nav> elements
          ----
            <article>                             //  separable content possible to distribute independently from the page (Forum / Blog / News)
          ----
            <section>                             //  thematic grouping of content (like chapters)
                                                     - it has typically a heading 
          ----    
            <aside>                               //  portion of the document which is indirectly related (side-bars, call-out boxes)
          ----                                   
            <footer>                              //  represents a footer to the document or the nearest section content (copyright, contant, sitemap informations)
                                                     - multiple allowed on a page
