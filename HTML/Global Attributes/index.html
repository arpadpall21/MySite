Description and Demonstration :

  • xml:lang / xml:base inherited from the XHTML specification
  • aira-* multiple attribute for assistive technology

- in HTML5 we can use any characters as Identifiers but it complicates thigs 

SYNTAX :    class="idName idName ..."                    // specifies one or more identifier(s) for the element    
            id="idName"                                  // specifies a unique identifier for the element     
            title="text"                                 // specify extra informations about the element (shown as toolpit)
            lang="langCode"                              // specify the language of the element's content W3C language codes [link]
            style="cssStyle"                             // specifies an inline style for the element 
            
                - idName : identifier must start with a letter it can contain a-zA-Z(letters) 0-9(digits) _(undersocre) -(hyphen)) (in HTML5 identifiers can use ANY characters except white space and cannot start with a digit)  
    
// id / class ----------------------------------------------------
-- HTML -- 
    <p id="@yd" class="myClass"> test_paragraph_1 </p>   // id must be unique 
    <p class="myClass"> test_paragraph_2 </p>            // same class identifiers identify multiple elements at once  
    <p class="myClass 2yourClass"> test_paragraph_3 </p> // element is identified by 2 different class ids 
    
-- CSS -- 
    #\@yd { color: green; }                              // escape character    // -! this makes kind of pointless and complicated to use weird identifier names 
    
// title / lang -------------------------------------------------- 
    <p title="myTitle"> this element has a title </p>
    <p lang="hu"> ez magyar nyelven van irval </p>
    
// style ---------------------------------------------------------

data- attribute

- the "data" attribute stores custom data (string only) which can be handeled through JS

- these informations are stored in the dataset property (object) (called DOMStringMap) of the element

- the property should contain only lowercase letters and dashes are converted to (camel-case JS=> camelCase) in JS

SYNTAX :    data-property="value" 

    // API --------------------------- 
            element.dataset                              // returns the DOMStringMap object of the element which holds the properties (and their values) set by the data global attribute   
            element.dataset.property                     // gets or sets the targeted property of the DOMStringMap object
    
    <p id="demo" data-demo-prop1="myValue" data-demo-prop2="2"> myParagraph </p>
<script>
    var element = document.getElementById("demo");
    element.dataset;                                     // -> {demoProp1:"myValue", demoProp2:"2"}
    typeof element.dataset.demoProp2;                    // -> string (stores only string)
// add new data property -------
    element.dataset.demoProp3 = true;
    element.dataset;                                     // -> {demoProp1:"myValue", demoProp2:"2", demoProp3:"true"}
</script>

tabindex attribute 

SYNTAX :    tabindex="nr"                                // defines the tabbing order of the element (not focusable elements become focusable in HTML5) 
                - nr : positiveValue - defines the tabbing order within a group having the tabindex attribute 
                       0             - browser defines the tabbing order 
                       negativeValue - element becomes focusable (not reachable by navigating with the "tab" button)
    
    <p tabindex="2"> element </p>                        -|
    <p tabindex="3"> element </p>                         | // this is a tabbing group element, (when the "Tab" button is hit these elements are the first to get focus on this page in the defined tabbing order!)    
    <p tabindex="1"> element </p>                        -|
    
    <p tabindex="0"> element </p>                        // element becomes focusable and the browser defines its tabbing order (document flow)
    <p tabindex="-1"> element </p>                       // element becomes focusable only! (not reachable by hitting the "Tab" or "Shift + Tab" buttons)
